# RSS Daily Brief

从配置的 RSS 源抓取最新内容，去重后输出结构化摘要供你阅读和总结。

## 工作流程

### 第一步：抓取数据
执行脚本获取原始 RSS 内容：
```bash
python3 ~/.hermes/skills/rss-reader/rss_reader.py
```

### 第二步：格式化输出
拿到脚本输出后，你需要按以下格式整理呈现：

**🔥 核心主题**（将相关新闻按主题聚类，每个主题写 2-3 句分析性叙述，用粗体强调关键信息，附 🔗 来源链接。产出 6-8 个主题。）

**🌐 开源社区**（GitHub Trending 相关条目，按排名列出）

**🤖 LLM 动态**（模型发布、工具平台、生态应用三个子类，各 3-5 条）

**💰 投融资快报**（3-5 条，注明金额和投资方）

**🛠️ 开发者工具**（3-5 条新工具/框架发布）

**📰 行业深度**（3-4 篇，写较长的分析摘要，可引用原文关键段落）

**📊 论文精选**（2-3 篇重要论文，包含：标题、链接、作者、核心发现、为什么重要）

**💡 编者观察**（500-600 字，把当天的故事串成趋势分析，有观点）

**输出要求：**
- 全部用中文
- 不要照搬 RSS 原文，用自己的话概括和分析
- 如果某个分类当天没有相关内容，跳过该栏
- 末尾附：共 N 条 | 来自 N 个源 | 更新时间

## RSS 源配置

源列表在 `~/.hermes/skills/rss-reader/config.yaml`，用户可随时编辑添加或删除源。

## 输出说明

脚本输出包含：
- 分类标题（商业、AI、论文、科技、深度报道、开发工具）
- 每条内容的标题、链接、简介（200字以内）
- 已自动去除 HTML 标签和重复内容（7天内已推送的不会重复出现）

## 注意事项

- 单个源抓取失败不影响其他源，错误会在输出末尾显示
- 去重记录保存在 `~/.hermes/rss-history.json`，超过 7 天自动清理
- 如需立即刷新，可删除 `~/.hermes/rss-history.json` 后重新运行
